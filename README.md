# Chauny X Skill

一个面向个人研究工作流的 X 监控 skill，用来低成本追踪固定博主的最新帖子。

它的设计目标很明确：

- ✅ 不依赖官方 X API
- ✅ 适合只盯少量固定作者
- ✅ 支持去重、落盘、生成 Markdown 报告
- ✅ 适合“人工浏览器采集 + 本地整理”的研究流程
- ✅ 成本低、结构简单、可自己维护

## 这是什么

`chaunyx-skill` 是一个浏览器驱动、人工节奏的 X 作者监控工具。

它不做广义爬虫，也不做大规模自动化抓取。它只解决一个实际问题：

> 我只想固定盯几个 X 博主，把他们最近的帖子抓下来，去重，整理成我能读的笔记。

## 能干什么

| 能力 | 状态 | 说明 |
| --- | --- | --- |
| 固定作者 watchlist | ✅ | 维护一组固定的 `username` |
| 人工采集后批量导入 | ✅ | 将浏览器里看到的帖子整理成 JSON 再导入 |
| 去重 | ✅ | 按 `post_id` 去重，避免重复写入 |
| JSONL 事件落盘 | ✅ | 方便后续做分析或二次处理 |
| Markdown 报告生成 | ✅ | 每次导入后自动更新一份报告 |
| 成本友好 | ✅ | 不依赖官方 X API 计费 |
| 广域热门发现 | ❌ | 这不是热点爬虫 |
| 自动发帖/互动 | ❌ | 永远保持只读 |
| 大规模后台常驻抓取 | ❌ | 不面向高频无人值守 |

## 适合谁

这套 skill 特别适合下面这类场景：

- ✅ 只关注少量作者，比如 3 到 20 个
- ✅ 希望低成本持续观察某些博主
- ✅ 可以接受“人工打开主页 + 半自动整理”
- ✅ 更看重稳定的个人研究流程，而不是大规模抓取
- ✅ 想把内容沉淀成自己的 Markdown / JSON 数据资产

## 不适合什么

如果你的目标是下面这些，这个仓库就不是最优选：

- ❌ 全站搜索
- ❌ 实时热门榜监控
- ❌ 高并发自动采集
- ❌ 大规模无头浏览器集群
- ❌ 企业级爬虫基础设施

## 工作流

整个流程非常简单：

| 步骤 | 你做什么 | skill 做什么 |
| --- | --- | --- |
| 1. 初始化 | 生成配置文件 | 创建默认 watchlist 配置 |
| 2. 检查配置 | 看作者列表是否正确 | 校验路径、作者数量、输出目录 |
| 3. 浏览器采集 | 打开作者主页，复制最近帖子 | 不直接抓浏览器，只接收标准 JSON |
| 4. 导入批次 | 运行 `ingest` | 去重、落 JSONL、更新 Markdown 报告 |
| 5. 后续重复 | 再次采集新帖子 | 自动跳过旧内容 |

## 命令

### 1. 初始化配置

```powershell
python scripts/x_manual_monitor.py init --config .\x-manual-monitor.json
```

### 2. 检查配置

```powershell
python scripts/x_manual_monitor.py check --config .\x-manual-monitor.json
```

### 3. 生成批次模板

```powershell
python scripts/x_manual_monitor.py make-batch-template .\batch.json --config .\x-manual-monitor.json
```

### 4. 导入帖子批次

```powershell
python scripts/x_manual_monitor.py ingest .\batch.json --config .\x-manual-monitor.json
```

## 输入格式

每条帖子都需要这些字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `post_id` | ✅ | X 帖子 ID |
| `author` | ✅ | 用户名 |
| `url` | ✅ | 帖子链接 |
| `published_at` | ✅ | 发布时间 |
| `text` | ✅ | 帖子正文 |
| `label` | 可选 | 展示名，不填则回退到作者名 |

示例：

```json
[
  {
    "post_id": "1899999999999999999",
    "author": "elonmusk",
    "label": "Elon Musk",
    "url": "https://x.com/elonmusk/status/1899999999999999999",
    "published_at": "2026-04-18T09:20:00Z",
    "text": "Example post text"
  }
]
```

## 输出结果

导入后会生成这些内容：

| 输出 | 位置 | 用途 |
| --- | --- | --- |
| 事件流 | `events/events.jsonl` | 保存结构化帖子数据 |
| 状态文件 | `state/state.json` | 记录已处理 `post_id` |
| Markdown 报告 | `reports/latest-report.md` | 快速阅读最新导入内容 |

## 使用建议

### 推荐做法

- ✅ 只监控最重要的作者
- ✅ 每天 1 到 2 次人工采集
- ✅ 每次只看最近 3 到 10 条帖子
- ✅ 统一存成 JSON 再导入
- ✅ 把 Markdown 报告纳入你自己的研究笔记流

### 不推荐做法

- ❌ 把它当全自动爬虫
- ❌ 试图覆盖大量作者
- ❌ 依赖首页 `For You`
- ❌ 混入无关回复、广告、置顶噪音

## 设计取舍

| 设计点 | 选择 | 原因 |
| --- | --- | --- |
| 数据源 | 浏览器人工采集 | 更适合低成本个人使用 |
| 自动化程度 | 半自动 | 保持简单和可控 |
| 存储格式 | JSONL + Markdown | 兼顾结构化和可读性 |
| 去重键 | `post_id` | 稳定、直接 |
| 目标范围 | 固定作者 | 避免噪音和复杂度膨胀 |
